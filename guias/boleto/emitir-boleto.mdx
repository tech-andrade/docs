Todos os tipos de boleto são criados pelo mesmo endpoint — `POST /baas/v1/charges`. O campo `type` define se é um boleto convencional (`BANKSLIP`) ou com QR Code Pix integrado (`BANKSLIP_PIX`).

<Note>
Sua conta precisa estar habilitada como **Cedente** antes de emitir boletos. Veja [Visão Geral](/guias/boleto/visao-geral) para detalhes sobre esse pré-requisito.
</Note>

## Passo a passo

<Steps>

<Step title="Emita o boleto">
O mínimo necessário é o `type`, `correlationId`, `yourNumber`, `dueDate`, `Amount` e os dados completos do `payer`.

<CodeGroup>
```bash cURL
curl --location 'https://apisandbox.delbank.com.br/baas/v1/charges' \
--header 'Content-Type: application/json' \
--header 'x-delbank-api-key: {{apiKey}}' \
--header 'x-delfinance-account-id: {{accountId}}' \
--data '{
    "type": "BANKSLIP",
    "correlationId": "{{$guid}}",
    "yourNumber": "PED-2024-001",
    "dueDate": "2025-12-30",
    "Amount": 150.00,
    "payer": {
        "name": "João Alves",
        "document": "07034346593",
        "email": "joao@email.com",
        "phone": { "prefix": "79", "number": "988669383" },
        "address": {
            "zipCode": "49010030",
            "publicPlace": "AV. RIO BRANCO",
            "neighborhood": "CENTRO",
            "number": "186",
            "complement": "SALA 602",
            "city": "ARACAJU",
            "state": "SE"
        }
    }
}'
```

A resposta traz `barCode` e `digitableLine` — use esses valores para exibir o boleto ao pagador ou gerar o PDF (veja [Gerenciar Boleto](/guias/boleto/gerenciar-boleto)).
</Step>

<Step title="Apresente o boleto ao pagador">
Para boletos convencionais (`BANKSLIP`), você tem duas opções:

- **Linha digitável** (`digitableLine`) — exiba em texto para o pagador copiar no app do banco.
- **PDF** — gere via `GET /baas/v1/charges/:correlationId/stream` para enviar por e-mail ou exibir na tela. Veja [Gerenciar Boleto](/guias/boleto/gerenciar-boleto).

Para `BANKSLIP_PIX`, a resposta também inclui `qrCode` (payload) e `qrCodeImageBase64`, permitindo que o pagador escolha entre boleto ou Pix.
</Step>

<Step title="Acompanhe o pagamento via Webhook">
Quando o boleto for pago, a API dispara um evento de webhook com os dados da liquidação. Configure a URL de webhook no painel do desenvolvedor para receber essas notificações em tempo real.
</Step>

</Steps>

## Boleto com QR Code Pix (BANKSLIP_PIX)

Para emitir um boleto que também pode ser pago via Pix, use `"type": "BANKSLIP_PIX"`. O endpoint e os campos são os mesmos — a resposta adiciona `qrCode` e `qrCodeImageBase64`.

--header 'Content-Type: application/json' \
--header 'x-delbank-api-key: {{apiKey}}' \
--header 'x-delfinance-account-id: {{accountId}}' \
--data '{
    "type": "BANKSLIP_PIX",
    "correlationId": "{{$guid}}",
    "yourNumber": "PED-2024-002",

## Adicionar desconto, multa e juros

Use os campos `discount`, `lateFine` e `latePayment` para configurar os encargos. Todos são opcionais.

```json
{
    "type": "BANKSLIP_PIX",
    "correlationId": "{{$guid}}",
    "yourNumber": "PED-2024-003",
    "dueDate": "2025-12-30",
    "Amount": 200.00,
    "payer": { "..." : "..." },
    "discount": {
        "type": "Fixed",
        "items": [
            { "date": "2025-12-10", "amount": 20.00 },
            { "date": "2025-12-20", "amount": 10.00 },
            { "date": "2025-12-28", "amount": 5.00 }
        ]
    },
    "lateFine": {
        "type": "Percentage",
        "date": "2025-12-31",
        "amount": 2.0
    },
    "latePayment": {
        "type": "Fixed",
        "date": "2025-12-31",
        "amount": 2.50
    }
}
```

| Campo | Quando aplicado | `type` aceito |
|---|---|---|
| `discount.items[]` | Antes das datas definidas (até 3 faixas) | `Fixed`, `Percentage` |
| `lateFine` | Uma vez, após `lateFine.date` | `Fixed`, `Percentage` |
| `latePayment` | Por período, após `latePayment.date` | `Fixed`, `Percentage` |

<Note>
O `discount` aceita até **3 itens** com datas diferentes — quanto antes pagar, maior o desconto. As datas devem ser anteriores ao `dueDate`.
</Note>

## Boleto com beneficiário final (para gateways)

Se você opera como gateway ou plataforma emitindo boletos em nome de terceiros, use `finalBeneficiary` para identificar quem de fato receberá o pagamento.

```json
{
    "type": "BANKSLIP",
    "correlationId": "{{$guid}}",
    "yourNumber": "PED-2024-004",
    "dueDate": "2025-12-30",
    "Amount": 300.00,
    "payer": { "..." : "..." },
    "finalBeneficiary": {
        "name": "Loja do João",
        "document": "12345678000195",
        "address": {
            "zipCode": "49000000",
            "publicPlace": "RUA PRINCIPAL",
            "neighborhood": "CENTRO",
            "number": "100",
            "city": "ARACAJU",
            "state": "SE"
        }
    }
}
```

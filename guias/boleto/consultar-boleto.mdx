Existem duas formas de consultar boletos: por `correlationId` (consulta individual) ou por período com paginação (consulta em lote para conciliação).

<Tip>
Prefira receber eventos via **webhook** para acompanhar pagamentos em tempo real, em vez de polling por consulta periódica. A consulta paginada é mais adequada para conciliação e auditoria.
</Tip>

## Consultar por correlationId

Retorna os dados completos de um boleto específico, incluindo os encargos configurados e o histórico de pagamentos.

<CodeGroup>
```bash cURL

--header 'x-delbank-api-key: {{apiKey}}' \
--header 'x-delfinance-account-id: {{accountId}}'
```

## Consulta paginada por período

Retorna uma lista de boletos criados dentro do intervalo de datas informado. Útil para conciliação financeira e geração de relatórios.

```bash cURL
curl --location 'https://apisandbox.delbank.com.br/baas/v1/charges?startDate=2025-01-01&endDate=2025-01-31&page=1&limit=50' \
--header 'x-delbank-api-key: {{apiKey}}' \
--header 'x-delfinance-account-id: {{accountId}}'
```

### Parâmetros de query

| Parâmetro | Tipo | Obrigatório | Descrição |
|---|---|---|---|
| `startDate` | `string` | Sim | Data inicial no formato `YYYY-MM-DD` |
| `endDate` | `string` | Sim | Data final no formato `YYYY-MM-DD`. Máximo: 1 ano de intervalo |
| `page` | `number` | Não | Página (começa em 1) |
| `limit` | `number` | Não | Itens por página — máximo 100 |

<Note>
O intervalo máximo permitido entre `startDate` e `endDate` é de **1 ano**. Para períodos maiores, faça múltiplas consultas.
</Note>

A resposta é um array de objetos com os mesmos campos da consulta individual, mas sem `payments` expandido.

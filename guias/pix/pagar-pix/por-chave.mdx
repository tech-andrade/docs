Quando você tem a chave Pix do recebedor (CPF, CNPJ, e-mail, telefone ou chave aleatória EVP), o fluxo acontece em dois passos: primeiro você consulta a chave para obter o `endToEndId` e os dados da conta destino, depois executa o pagamento com esse identificador.

## Passo a passo

<Steps>

<Step title="Inicialize o pagamento consultando a chave">
Chame o endpoint de inicialização informando a chave Pix do recebedor. A API consulta o DICT (Diretório de Identificadores de Contas Transacionais) do Banco Central e retorna os dados da conta vinculada.

<CodeGroup>
```bash cURL
curl --location 'https://apisandbox.delbank.com.br/baas/api/v2/pix/dict/payment-initialization' \
--header 'Content-Type: application/json' \
--header 'x-delbank-api-key: {{apiKey}}' \
--header 'x-delfinance-account-id: {{accountId}}' \
--data '{
    "key": "+5511999975734"
}'
```
```json Response
{
    "endToEndId": "E38224857202412201244TP05NXG36QK",
    "key": "+5511999975734",
    "beneficiary": {
        "number": "29823",
        "branch": "1",
        "type": "CURRENT",
        "participant": {
            "name": "DELCRED SCD S.A.",
            "ispb": "38224857"
        },
        "holder": {
            "name": "HOMOLOGACAO INTEGRACAO API",
            "document": "***876977**",
            "type": "NATURAL"
        }
    },
    "keyBelongsHolder": false
}
```
</CodeGroup>

O `endToEndId` retornado é de uso único e tem validade curta — use-o imediatamente na etapa seguinte.

<Tip>
Passe `holderDocument` no corpo da requisição se quiser verificar se a chave pertence a um CPF ou CNPJ específico. O campo `keyBelongsHolder` na resposta indica o resultado da verificação — útil para confirmar que o pagador está enviando para a pessoa certa.
</Tip>
</Step>

<Step title="Confirme os dados do recebedor">
Antes de executar o pagamento, exiba os dados do `beneficiary` retornado para que o usuário confirme que está enviando para a pessoa certa. Isso evita erros e é uma boa prática de UX para qualquer produto financeiro.

Os dados do titular aparecem com o documento parcialmente mascarado — isso é comportamento padrão do DICT por privacidade.
</Step>

<Step title="Execute a transferência">
Com o `endToEndId` em mãos, execute o pagamento. Use `initiationType: "KEY"` para indicar que a iniciação foi feita via chave Pix.

<CodeGroup>
```bash cURL
curl --request POST \
  'https://apisandbox.delbank.com.br/baas/api/v2/transfers' \
  --header 'IdempotencyKey: {{$guid}}' \
  --header 'Content-Type: application/json' \
  --header 'x-delbank-api-key: {{apiKey}}' \
  --header 'x-delfinance-account-id: {{accountId}}' \
  --data '{
    "amount": 50.00,
    "description": "Pagamento ref. pedido #1234",
    "endToEndId": "E38224857202412201244TP05NXG36QK",
    "initiationType": "KEY"
}'
```
```json Response
{
    "id": "0fc7821d-b488-4268-88c0-a2bc443d046e",
    "endToEndId": "E38224857202412201244TP05NXG36QK",

    "status": "PIX_PROCESSING",
    "type": "PIX_KEY",

    "createdAt": "2024-12-20T12:44:00.251Z",
    "description": "Pagamento ref. pedido #1234",
    "payer": {
        "number": "31712",
        "branch": "0001",
        "type": "CURRENT",
        "holder": {
            "document": "32752023000149",
            "name": "MINHA EMPRESA LTDA",
            "type": "LEGAL"
        },
        "participant": {
            "name": "DELBANK",
```

O status inicial é `PIX_PROCESSING`. O status final (`PIX_EFFECTIVE` ou `PIX_ERROR`) é entregue via webhook — não use polling.

<Warning>
O `IdempotencyKey` no header é obrigatório. Envie um GUID único por requisição. Se a mesma chamada for reenviada com o mesmo `IdempotencyKey`, a API retornará o resultado original sem criar um pagamento duplicado. Veja [Idempotência](/iniciacao/idempotencia).
</Warning>
</Step>

</Steps>

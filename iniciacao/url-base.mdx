Todas as chamadas às APIs Delfinance devem ser realizadas sobre HTTPS, com as camadas de segurança apropriadas conforme o ambiente.

## URLs por ambiente

<CodeGroup>

```bash Sandbox
https://apisandbox.delbank.com.br
```

```bash Produção
https://api.delbank.com.br
```

</CodeGroup>

## Autenticação por headers

No modelo atual, a autenticação é feita com os headers abaixo:

| Cabeçalho | Obrigatório | Descrição |
| --- | --- | --- |
| `x-delbank-api-key` | Sim | Identifica a integração e autentica a organização |
| `x-delfinance-account-id` | Sim | Identifica a conta acessada |

## Versionamento

As APIs seguem versionamento via path. A versão atual é `v1`:

```text
https://api.delbank.com.br/v1/{recurso}
```

<Info>
  Mudanças com quebra de compatibilidade sempre resultam em uma nova versão. Melhorias retrocompatíveis são adicionadas à mesma versão.
</Info>

## Formato das requisições

- **Protocolo:** HTTPS + TLS 1.2 ou superior
- **Autenticação por headers:** obrigatória em todos os ambientes
- **mTLS:** obrigatório em Produção
- **Allowlist de IPs:** obrigatória em Produção
- **Content-Type:** `application/json`
- **Encoding:** UTF-8

## Cabeçalhos padrão

| Cabeçalho | Obrigatório | Descrição |
| --- | --- | --- |
| `x-delbank-api-key` | Sim | Chave de API da integração |
| `x-delfinance-account-id` | Sim | Identificador da conta |
| `Content-Type` | Sim | `application/json` |
| `IdempotencyKey` | Em operações de escrita | Chave única de idempotência |
| `X-Request-Id` | Recomendado | UUID para rastrear a requisição |

O recebimento de Pix envolve a criação de uma cobrança, a apresentação ao pagador e a confirmação assíncrona via webhook.

## Fluxo passo a passo

<Steps>
  <Step title="Cliente cria a cobrança">
    Sua aplicação chama a API para criar uma cobrança Pix, informando valor, descrição e dados do pagador (opcional).
  </Step>
  <Step title="DelFinance gera os dados de pagamento">
    Recebemos a solicitação e retornamos o QR Code (BR Code), o `pix_copy_paste` e o `transaction_id`.
  </Step>
  <Step title="Pagador realiza o Pix">
    O pagador escaneia o QR Code ou copia e cola no app do seu banco e conclui o pagamento.
  </Step>
  <Step title="DelFinance identifica o pagamento">
    O SPI confirma o crédito. A DelFinance atualiza o status da transação para `confirmed`.
  </Step>
  <Step title="Cliente recebe webhook">
    Enviamos um evento `pix.received` ao endpoint cadastrado.
  </Step>
  <Step title="Consulta opcional">
    Se necessário, seu backend pode consultar a transação para reconciliação.
  </Step>
</Steps>

## Diagrama

```text
┌──────────┐      ┌────────────┐      ┌────────────┐      ┌──────────┐
│  Cliente │─────▶│ DelFinance │─────▶│    SPI     │◀─────│ Pagador  │
│ (backend)│      │            │      │  (Bacen)   │      │  (banco) │
└────┬─────┘      └─────┬──────┘      └─────┬──────┘      └──────────┘
     │  1. cria cobrança │                    │
     │──────────────────▶│                    │
     │  2. QR Code + txId│                    │
     │◀──────────────────│                    │
     │                   │                    │  3. Pix enviado
     │                   │◀───────────────────│
     │                   │  4. crédito confirmado
     │  5. webhook pix.received                │
     │◀──────────────────│                    │
     │  6. GET /v1/pix/transactions/{id}       │
     │──────────────────▶│                    │
```

## Exemplo — criar cobrança

<CodeGroup>

```bash cURL
curl -X POST https://api.sandbox.delfinance.com.br/v1/pix/charges \
  --cert ./certificado.crt --key ./chave-privada.key \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -H "X-Idempotency-Key: $(uuidgen)" \
  -d '{
    "amount": 150.75,
    "description": "Pagamento do pedido #1234",
    "expires_in": 3600
  }'
```

```javascript Node.js
import { randomUUID } from 'crypto';

const { data } = await axios.post(
  'https://api.sandbox.delfinance.com.br/v1/pix/charges',
  {
    amount: 150.75,
    description: 'Pagamento do pedido #1234',
    expires_in: 3600,
  },
  {
    httpsAgent: agent,
    headers: {
      Authorization: `Bearer ${token}`,
      'X-Idempotency-Key': randomUUID(),
    },
  }
);
```

```csharp C#
using var client = new HttpClient(handler);
client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", token);
client.DefaultRequestHeaders.Add("X-Idempotency-Key", Guid.NewGuid().ToString());

var body = new {
    amount = 150.75,
    description = "Pagamento do pedido #1234",
    expires_in = 3600
};

var response = await client.PostAsJsonAsync(
    "https://api.sandbox.delfinance.com.br/v1/pix/charges",
    body
);
```

</CodeGroup>

```json
  "status": "created",
  "amount": 150.75,
  "description": "Pagamento do pedido #1234",
  "qr_code": "00020126360014BR.GOV.BCB.PIX0114+5511999999999...",
  "qr_code_image": "https://cdn.delfinance.com.br/qr/pix_tx_123456.png",
  "pix_copy_paste": "00020126360014BR.GOV.BCB.PIX...",
  "expires_at": "2026-01-01T13:00:00Z",
  "created_at": "2026-01-01T12:00:00Z"
}
```

<Warning>
  Nunca considere a cobrança paga com base apenas na resposta síncrona. Confirme sempre via [webhook](/pix/webhooks) ou consulta.
</Warning>
